# Git Commit Rules

## Commit Message Format

```
<type>: <subject>

<body (optional)>
```

## Type

- `feat`: new feature
- `fix`: bug fix
- `refactor`: refactoring
- `style`: code formatting
- `test`: add/modify tests
- `docs`: documentation updates
- `chore`: build/config changes

## Example

```
feat: Add user profile screen with Compose

- Implement UserProfileScreen composable
- Add UserProfileViewModel with StateFlow
- Integrate with UserRepository
```
